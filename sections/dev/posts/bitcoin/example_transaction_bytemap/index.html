    inputs

    Let's look closely into the inputs. The raw input data is 01eb3ae38f27191aa5f3850dc9cad00492b88b72404f9da135698679268041c54a0100000000ffffffff. If you parse it:

    count           : 01
prev output txid: eb3ae38f27191aa5f3850dc9cad00492b88b72404f9da135698679268041c54a
  flip_bytes -> 4ac541802679866935a19d4f40728bb89204d0cac90d85f3a51a19278fe33aeb
prev output ind : 01000000
  flip_bytes -> 00000001
  -> check 01th vout from the fetched transaction using the txid.
length          : 00
sequence        : ffffffff

    Note that the prev output txid and prev output index are flipped due to the difference in endianness.

    Scripts

    Here are shell scripts, which might be useful.

    To flip byte orders:

    #!/bin/sh
# flip_bytes.sh - Reverse byte order of a hex string
hex="$1"
echo "$hex" | fold -w2 | tac | tr -d "\n"
echo

    To convert hex string to decimal

    #!/bin/sh
# hex_to_dec.sh - Convert hex string to decimal
hex="$1"
printf "%d\n" "0x$hex"
