    Bitcoin regtest 101

    References

    This post is a personal note for commands. To see more detailed explanation, visit:

    • blockchain commons
    • from forum

    bitcoin-cli

    Config

    • Run with bitcoind -regtest -fallbackfee=0.00001 -maxtxfee=0.1
      • the fallbackfee and maxtxfee setting will prevent Fee estimation failed error
    • Or modify ~/.bitcoin/bitcoin.conf like below and run with bitcoind
      regtest=1

[regtest]
fallbackfee=0.00001
maxtxfee=0.1
server=1
printtoconsole=1

    The regtest state will be saved in ~/.bitcoin/regtest directory To start a fresh regtest, simply remove the directory with rm -rf regtest

    If you set the regtest=1 in the bitcoin.conf, you do not need to add the -regtest argument for bitcoin-cli. Otherwise, you have to use the cli like bitcoin-cli -regtest ...

    Wallet

    A Wallet is needed to generate blocks.

    bitcoin-cli -regtest -named createwallet wallet_name="regtest_desc_wallet" descriptors=true
bitcoin-cli createwallet <wallet name>
bitcoin-cli loadwallet <wallet name>
bitcoin-cli unloadwallet
bitcoin-cli getnewaddress <label> <address_type(optional)>
bitcoin-cli getnewaddress // a bech32 address starting with "bcrt"
bitcoin-cli getnewaddress "" legacy
bitcoin-cli getnewaddress "segwit address" p2sh-segwit

    Generate Blocks

    Note that a block must have 100 confirmations before the reward can be spent. Before that the balance will not show the rewards.

    Also note that the difference rewards curve to the mainnet.

    bitcoin-cli -generate 101 // mine 101 blocks -> the reward from first block can be spent
bitcoin-cli getbalance // will show the first rewards: 50


bitcoin-cli generagtetoaddress <number_of_blocks> <address>

    Use the Regtest

    Transactions

    $ bitcoin-cli -regtest sendtoaddress [address] [amount_to_send]

    The output is the transaction hash, which can be verified using gettransaction

    bitcoin-cli gettransaction <tx_hash>

    To finalize, create blocks

    bitcoin-cli -generate 6

    create raw transaction

    You can send bitcoin with following steps:

    1. create raw transaction
    • specify the outputs you want to spend
    • specify the destination and amount (the left-over as fee for miners)
    1. sign the raw transaction
    2. send the signed transaction
    // This will return raw transaction <hex>
> bitcoin-cli createrawtransaction
> bitcoin-cli createrawtransaction [{"txid":"hex","vout":n,"sequence":n},...] [{"address":amount,...},{"data":"hex"},...] ( locktime replaceable )
> bitcoin-cli createrawtransaction \
  '[{"txid":"<txid>","vout":<vout>}]' \
  '{"<dest_address>": <amount>}'

// sign it
// if the output to spend is received by address created by `getnewaddress` from the current wallet
// it can sign it without trouble like below
> bitcoin-cli signrawtransactionwithwallet <hex>

// send the signed transaction
// this will return <txid>
> bitcoin-cli sendrawtransaction <hex>

// check the transaction
> bitcoin-cli gettransaction <txid>
> bitcoin-cli getrawtransaction <txid> true
> bitcoin-cli getmempoolentry <txid>
