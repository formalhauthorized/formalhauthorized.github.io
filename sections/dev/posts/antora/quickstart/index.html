    Quick Start Antora

    13 August 2025

    Environment

    For the ease of setting, using a docker container is a good start. The Dockerfile below is an example environment.

    # syntax=docker/dockerfile:1.6

####################################################
### ARGs
####################################################

ARG VARIANT="trixie"

####################################################
### base
####################################################

# [Choice] Debian version (use bullseye on local arm64/Apple Silicon): bookworm, bullseye, buster
FROM debian:${VARIANT} AS base-image

RUN apt-get update && export DEBIAN_FRONTEND=noninteractive \
    && apt-get -y install --no-install-recommends \
    curl \
    ca-certificates \
    gnupg \
    bash-completion \
    git \
    jq \
    python3-pip \
    python3-venv \
    sudo \
    unzip \
    wget \
    vim \
    tmux \
    tree \
    ssh \
    && rm -rf /var/lib/apt/lists/*

####################################################
### user
####################################################

FROM base-image AS toolbox

# Add a user with passwordless sudo
ARG USERNAME=dev
RUN useradd -m "${USERNAME}" && \
    usermod -aG sudo "${USERNAME}" && \
    echo "${USERNAME} ALL=(ALL) NOPASSWD: ALL" >> /etc/sudoers

##### user-level setup follows
##### Things should be installed in $HOME from now on
USER "${USERNAME}"
WORKDIR /home/"${USERNAME}"
ENV HOME="/home/${USERNAME}"
ENV PATH="${PATH}:${HOME}/.local/bin"

# Install NVM
# inspired by https://raw.githubusercontent.com/devcontainers/features/main/src/node/install.sh
RUN curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
ENV NVM_DIR="${HOME}/.nvm"

# Install Node, get Node version, update PATH and install global npm packages
RUN . "$NVM_DIR/nvm.sh" && \
    nvm install --lts && \
    nvm alias default lts/* && \
    nvm use default && \
    node_version=$(node --version) && \
    # echo "PATH=\"${HOME}/.nvm/versions/node/$node_version/bin:${PATH}\"" >> "${HOME}/.bashrc" && \
    . "${HOME}/.bashrc" && \
    npm install --omit=dev --global --force pnpm && \
    # Install Yarn
    npm install -g yarn

### ANTORA: https://docs.antora.org/antora/latest/install-and-run-quickstart/
RUN \
    . "$NVM_DIR/nvm.sh" && \
 nvm install --lts && \
 nvm alias default 22

    Put the code into Dockerfile and build it with docker build -t <tag> .. Run it with docker run -p 3000:3000 -it antora_101 bash, note the port is mapped from 3000 to 3000.

    Note that you have to run . ~/.nvm/nvm.sh to use node or npm. Maybe put it into ~/.bashrc might be a good idea.

    make a package

    mkdir docs-site && cd docs-site
node -e "fs.writeFileSync('package.json', '{}')" && npm i -D -E antora
npx antora -v

    Playbook with a local source

    git initialize the docs-site

    git init
git commit --allow-empty -m init

    Putting some docs

    mkdir -p modules/ROOT/pages

    modules/ROOT/pages/index.adoc

    = Welcome!

You are home.

    modules/ROOT/pages/new-page.adoc

    = New Page

You've found a new page.

    modules/ROOT/nav.adoc

    * xref:new-page.adoc[]

    antora.yml

    name: project-name
version: ~
title: Project Name
nav:
- modules/ROOT/nav.adoc

    site:
  title: Docs Site
  start_page: project-name::index.adoc
content:
  sources:
  - url: .
    branches: HEAD
ui:
  bundle:
    url: https://gitlab.com/antora/antora-ui-default/-/jobs/artifacts/HEAD/raw/build/ui-bundle.zip?job=bundle-stable
    snapshot: true

    To build:

    npx antora antora-playbook.yml

    serve

    To serve the build/site, use the following command. By default, it will serve to the port 3000, and you can access it on your browser with localhost:3000.

    npx serve build/site

    If you want to specify the port (for example, 8080):

    npx serve build/site -l 8080
